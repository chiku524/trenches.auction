## Learned User Preferences

- On Windows, install and use Solana CLI from WSL; the official installer does not support Git Bash/MSYS.
- In bash, quote npm script arguments that contain `#` (for example cNFT mint names) so `#` is not treated as starting a comment.
- cNFT artwork should read as creature-like and realistically shaded (anime–3D / PBR); keep the deterministic 2D preview and the web 3D viewer stylistically aligned.
- For the web 3D creature viewer, when archetypes read as too generic or blob-like, the user prefers rebuilding bodies and shapes from scratch rather than only small tweaks.

## Learned Workspace Facts

- The `cnft/` package runs Metaplex Bubblegum cNFT flows using `KEYPAIR_PATH`, `CLUSTER`, `METADATA_BASE_URL` aligned with the Worker `PUBLIC_BASE_URL`, and optional `RPC_URL` for devnet. Scripts run under Windows Node with the keypair JSON path; Solana CLI may live only in WSL. npm peer warnings between nested Metaplex packages and `@metaplex-foundation/umi` are common and usually safe to ignore unless something fails at runtime.
- `register-token` needs `WORKER_URL` and `ADMIN_SECRET` matching the Worker `ADMIN_SECRET` secret; admin routes compare `x-admin-secret`; the value cannot be read back from Cloudflare after storage—set or rotate with `wrangler secret put` and mirror locally.
- `create-tree` uses Bubblegum V2; minting must call `mintV2`, not `mintV1`, or Bubblegum returns `UnsupportedSchemaVersion` (6003).
- Devnet transactions require a funded payer; public devnet airdrops are often rate-limited—use faucets or other funding paths.
- The `web/` Vite app supports optional `VITE_API_BASE` so local UI dev can call a separate `wrangler dev` origin (for example `http://127.0.0.1:8787`); when the deployed UI and Worker share one host, leave it unset and use relative `/v1/...` URLs.
- Admin **Collection setup** in the `web/` app includes **Large collection**: target supply (presets 100 / 1k / 10k), batch size, and **Mint toward target** for repeated Worker batch mints until the goal or limits; large runs take a long time and need the tab kept open.
- The Worker supports server-assisted cNFT minting (`/v1/mint/cnft` with a signed challenge), records mints and tree-related state in D1, and serves a DAS-backed gallery for the configured Merkle tree—alongside the `cnft/` CLI and `register-token`. `GET /v1/cnft/status` includes `mintAuthorityAddress` (from `CNFT_MINT_KEYPAIR`) for devnet funding; the `web/` status panel shows it. Server tree/mint uses a long RPC confirm window, `skipPreflight` on send, and bounded `getSignatureStatuses` polling on blockhash/confirm failures (capped for Cloudflare subrequest limits). `GET /v1/cnft/preview/:mint` serves R2 `images/.../preview.png` when present, otherwise deterministic SVG from `buildCreaturePreviewSvg` in `src/cnft-creature-art.ts` (species archetypes, biome palette, trait-driven shading); gallery items and Metaplex `image` use this route.
- GitHub Actions (`.github/workflows/deploy-cloudflare.yml`) deploys on push to `main` via `npm ci`, `npm run build:web`, and `./node_modules/.bin/wrangler deploy` with secret `CLOUDFLARE_API_TOKEN`; `account_id` lives in `wrangler.jsonc`. Error 10021 (D1 binding "failed to generate") is often transient or platform-side—retry; ensure the token can edit D1, Workers, and R2 as needed. Error 10000 ("Authentication error") means the token lacks **Workers Scripts Edit** or **Workers Builds Configuration Edit**, or has zone-only scope; prefer a **user** token (Profile → API Tokens → "Edit Cloudflare Workers" template) over account-owned for CI—account tokens may also produce spurious `[9106]` on `/memberships`.
- Root `package-lock.json` must match `package.json` for `npm ci` in CI; if installs fail on missing transitive dependencies, regenerate the lockfile with `npm install` and commit the update.
- `CNFT_MINT_KEYPAIR` (JSON 64-byte array) and `CNFT_RPC_URL` are **Cloudflare Worker secrets** (set via `wrangler secret put`, not GitHub); public Solana RPC endpoints (`api.*.solana.com`) return 403 from Workers datacenter IPs—use a dedicated provider URL (Helius, QuickNode, Alchemy, Triton) with an API key on the same cluster as the tree.
- Live/environmental traits from `computeLiveMetadataTraits` are recomputed on each metadata read (not stored in D1), so JSON metadata reflects the current time and server rules on fetch unless clients cache responses.
- Per-archetype 3D meshes for the trench viewer live in `web/src/trench3d/CreatureByArchetype.tsx`; `@react-three/drei` `RoundedBox` expects materials as children of `RoundedBox` (not siblings). Shell-heavy archetypes (for example Coral Hermit and Pressure Snail) have used lathe geometry for shells with `RoundedBox` body masses. Verify `web/` changes with `npm run build` in `web/`.
